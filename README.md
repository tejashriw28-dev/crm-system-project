# Salesforce CRM Implementation Project

## Role
Junior Business Analyst & Project Coordinator

## Business Context
A mid-sized technology company was using Excel and emails to manage customer leads. This caused lead loss, delayed follow-ups, and poor reporting. The company decided to implement Salesforce CRM for 50 sales representatives.

## Project Objective
To design, configure, and roll out a Salesforce-based CRM system that improves lead tracking, sales performance, and customer follow-up efficiency.

## Tools Used
- Salesforce
- Jira
- Microsoft Excel
- Agile / Scrum
- Stakeholder Meetings
- Process Documentation

## Key Responsibilities
- Gathered and documented business requirements  
- Coordinated between sales, IT, and CRM vendor  
- Created Jira stories and sprint plans  
- Tracked risks, timelines, and deliverables  
- Supported testing and user training  

## Project Deliverables
- Business Requirements Document  
- Stakeholder Register  
- Jira Sprint Board  
- Risk Register  
- Implementation Timeline  
- User Acceptance Testing (UAT) Plan  

## Outcome
The CRM system improved lead visibility, reduced response time, and enabled management to track sales performance in real time.
